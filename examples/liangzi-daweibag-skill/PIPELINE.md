# 网红语录收集与整理技术方案

> 如何从零开始，将一个网红的公开内容系统性地收集、转录、提炼为结构化的认知框架。

本文档记录了「良子大胃袋.skill」的完整制作流程，可复用于任何公开内容创作者。

---

## 总览

```
视频/音频采集 → 语音转录 → 语录提取 → 认知框架提炼 → Skill 文件生成
```

整个流程分 5 个阶段，所需工具均为开源免费。

---

## 阶段一：素材采集

### 1.1 视频/音频下载

**工具**：[yt-dlp](https://github.com/yt-dlp/yt-dlp)（支持 B站、抖音、YouTube 等几百个平台）

```bash
# 安装
pip install yt-dlp

# 下载 B站视频的音频（只要音频，省空间）
yt-dlp -f "bestaudio" -o "video1.%(ext)s" "https://www.bilibili.com/video/BV1X8QTBiEnc"

# 下载整个视频（含画面）
yt-dlp -o "video1.%(ext)s" "https://www.bilibili.com/video/BV1X8QTBiEnc"
```

### 1.2 素材选择优先级

不是所有视频都值得转录。优先选择：

| 优先级 | 类型 | 原因 |
|--------|------|------|
| P0 | 核心内容（如吃播、专业领域视频） | 展示核心人设和标志性行为 |
| P0 | 认知转变期内容（如减肥宣言、重大决定） | 捕捉思维转折和决策逻辑 |
| P1 | 与粉丝/他人的长对话、Q&A、直播 | 展示即兴思维和真实性格 |
| P1 | 早期内容 | 了解性格底色和成长轨迹 |
| P2 | 争议回应 | 面对压力时的策略和真实情绪 |
| P2 | 日常生活片段 | 捕捉非表演状态的真实表达 |

**建议**：至少 5-8 个视频，覆盖不同时期和场景，目标一手素材量 5 万字以上。

### 1.3 二手素材搜索

除了一手视频，还需要搜集媒体报道和公众评价作为补充：

- **百度百科**：基本信息和时间线
- **新闻报道**：深度人物特写（新京报、腾讯新闻等）
- **知乎/微博**：公众讨论和多角度评价
- **行业分析文章**：对其商业模式/影响力的分析

## 阶段二：语音转录

### 2.1 推荐方案：mlx-whisper（Mac Apple Silicon）

在 M 系列 Mac 上，使用 mlx-whisper 可以利用 GPU 加速，速度是 CPU 版的 **5-10 倍**。

```bash
# 安装
pip install mlx-whisper

# 转录（中文）
python -c "
import mlx_whisper
result = mlx_whisper.transcribe('video1.m4a', path_or_hf_repo='mlx-community/whisper-medium-mlx', language='zh')
with open('video1.txt', 'w') as f:
    f.write(result['text'])
"
```

**实测速度**（M4 Pro, 48GB）：

| 文件大小 | 音频时长 | 转录耗时 |
|---------|---------|---------|
| 5.6MB | ~5分钟 | ~40秒 |
| 30MB | ~30分钟 | ~2分30秒 |
| 32MB | ~30分钟 | ~3分钟 |

### 2.2 备选方案：OpenAI Whisper（通用，CPU）

无 Apple Silicon 时使用：

```bash
# 安装
pip install openai-whisper

# 转录
whisper video1.m4a --model medium --language zh --output_dir ./transcripts/ --output_format txt
```

**注意**：CPU 模式下 30 分钟音频可能需要 20-30 分钟。

### 2.3 模型选择

| 模型 | 大小 | 中文效果 | 推荐场景 |
|------|------|---------|---------|
| tiny | 39M | 差 | 快速预览 |
| base | 74M | 一般 | 短视频快速转录 |
| small | 244M | 可用 | 对话较清晰的视频 |
| **medium** | **769M** | **较好** | **推荐：中文内容的最佳性价比** |
| large-v3 | 1550M | 最好 | 口音重/噪音大的场景 |

## 阶段三：语录提取与分类

### 3.1 人工/AI 辅助筛选

转录完成后，需要从原始文本中提取有价值的内容。可以使用 Claude 辅助：

```
请从以下转录文本中提取：
1. 反复出现的口头禅和标志性台词
2. 表达个人观点/价值观的语句
3. 面对特定场景（拒绝、批评、困难）的反应模式
4. 自创术语或独特的表达方式
5. 展示决策逻辑的关键对话

转录文本：
[粘贴转录内容]
```

### 3.2 分类存储结构

提取的内容按以下维度归档：

```
references/research/
├── 01-writings.md        # 核心语录与反复论点
├── 02-conversations.md   # 长对话与即兴思维
├── 03-expression-dna.md  # 表达风格DNA分析
├── 04-external-views.md  # 外部视角与争议
├── 05-decisions.md       # 决策记录与行为模式
└── 06-timeline.md        # 人物时间线
```

### 3.3 素材清单维护

每处理一个新素材，都在素材清单中记录来源、可信度、提取的关键内容，方便追溯和后续迭代。

## 阶段四：认知框架提炼

这是最核心的步骤——从语录和行为中抽象出可复用的认知模型。

### 4.1 心智模型提炼

每个心智模型应包含：

```markdown
### 模型名称（英文概括）

**一句话**：用一句话概括这个思维框架

**证据**：
- 来自一手素材的直接引用（标注来源）
- 多个场景中的一致行为模式

**应用**：
- 这个框架可以用在什么场景
- 对用户有什么具体帮助

**局限**：
- 这个框架在什么场景下不适用
- 可能的副作用
```

### 4.2 质量标准

- **有据可查**：每个模型至少有 2 个独立来源支持
- **可迁移**：不只是描述行为，而是提炼出可以应用于其他场景的思维方式
- **有局限**：诚实标注每个模型的适用边界
- **不编造**：对不确定的推断明确标注"推断"

### 4.3 诚实边界

必须明确标注：
- 一手素材量和可信度
- 哪些内容是直接引用，哪些是推断
- Skill 能做什么，不能做什么
- 信息截止时间

## 阶段五：Skill 文件生成

### 5.1 SKILL.md 结构

```markdown
---
name: skill-name
description: |
  一句话描述。素材来源。核心模型数量。
  触发词：「关键词1」「关键词2」
---

# 角色名 视角

## 角色扮演规则
## 身份卡
## 核心心智模型（每个含证据、应用、局限）
## 决策启发式
## 表达DNA（口头禅、句式、幽默方式、禁忌）
## 时间线
## 价值观与反模式
## 内在张力
## 诚实边界
## 调研来源
```

### 5.2 发布

```bash
# 初始化 Git 仓库
git init && git add . && git commit -m "Initial commit"

# 推送到 GitHub
gh repo create your-username/skill-name --public --push

# 其他用户安装
npx skills add your-username/skill-name
```
